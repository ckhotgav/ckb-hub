# 軟體設定檔急救指南：設定不見、軟體打不開怎麼辦？

當我們在學習寫程式時，經常會安裝各種外掛（例如新的 AI 模型 LiteLLM、擴充套件等）。但有時候一安裝完，**「原本的設定全部不見了」**，甚至**「軟體直接崩潰，完全打不開」**。

別慌！這通常不是軟體壞了，而是「設定檔（Config File）」出了小狀況。這篇教學將以 OpenCode 為例，教你如何化身軟體名偵探，用 4 個簡單的步驟自己把設定救回來！

---

## 為什麼設定會突然不見？

開發工具通常會把你的設定（例如背景顏色、快捷鍵、AI 模型清單）存在一個純文字檔裡（通常是 `.json` 格式）。
當你用指令安裝新外掛時，安裝程式可能會「太過暴力」，直接把原本的設定檔清空，或是用新設定把舊設定覆蓋掉。這時候軟體讀不到舊資料，你就會覺得「設定全都不見了」。

---

## 第一招：找出設定檔的「藏身處」

在 Windows 系統中，軟體的設定檔通常不會跟主程式放在一起，而是藏在你的「使用者專屬資料夾」裡。最常見的兩個藏寶地是：

1. **AppData 資料夾**（例如 VS Code）
2. **.config 資料夾**（開源軟體最愛，例如 OpenCode）

**🔍 實作步驟：**
1. 打開 Windows 檔案總管。
2. 在上方的「網址列」點一下，輸入 `%USERPROFILE%\.config` 然後按 Enter。
3. 你會看到裡面有一個 `opencode` 資料夾，點進去，這就是 OpenCode 所有設定檔的家！

---

## 第二招：看「檔案大小」，找出殘存的備份

進入 `opencode` 資料夾後，你可能會看到兩個長得很像的檔案：
* `opencode.jsonc`（系統真正會去讀取的檔案）
* `opencode.json`（可能是舊版或備份檔）

**🔍 實作步驟：**
注意看它們的**「檔案大小（容量）」**！
* 如果你發現 `opencode.jsonc` 只有非常小的 **50 Bytes**（代表被清空了）。
* 但旁邊的 `opencode.json` 卻有 **1 KB (1000 多 Bytes)**。

**💡 破案：** 
這代表你原本豐富的設定並沒有消失！只是軟體讀錯了檔案，或是安裝程式不小心把主檔案清空了。
**修復方法：** 用記事本打開容量大的檔案，把裡面的文字「全部複製」，貼到那個容量小的 `opencode.jsonc` 裡面，設定就回來啦！

---

## 第三招：破解「預設值消失」的秘密

有時候設定找回來了，但你發現**「內建的預設選項（例如 OpenAI、預設 Local 伺服器）全都不見了，只剩下你剛裝的 LiteLLM」**。

**🔍 運作邏輯：**
在設定檔的設計中，有一種規則叫做**「覆寫（Override）」**。
* **預設狀態：** 如果你不寫提供商（Provider），系統就會自動給你「內建全餐」。
* **自訂狀態：** 如果設定檔裡面出現了 `"provider": { "litellm-local": {...} }`，系統就會想：「喔！主人要自己指定選單，那我就『只』提供主人寫上的選項。」

**💡 破案：**
因為你的選單裡只寫了 LiteLLM，內建全餐就被隱藏了。
**修復方法：** 最簡單暴力的作法，就是把 `"provider": { ... }` 這一大塊**完全刪除**！系統找不到自訂清單，就會乖乖把內建全餐（預設模型）還給你。

---

## 第四招：終極檢查！為什麼軟體會完全當機？

如果你修改了設定檔，結果軟體連開都開不起來，直接跳出 `Unexpected server error` 或直接閃退，**99% 是你的 JSON 語法寫錯了！**

設定檔非常脆弱，它有嚴格的語法規定：
1. 每一個大括號 `{` 都要配一個 `}`。
2. 屬性之間必須用逗號 `,` 隔開。
3. 最後一個屬性後面**不可以**有逗號。

如果你在刪除或修改時，不小心多刪了一個 `}`，軟體就會因為「看不懂這篇文章」而當機。

**💡 破案與修復：**
1. 不要用眼睛檢查，人眼很容易漏看。
2. 把你的設定檔文字複製起來。
3. 打開瀏覽器搜尋 **「JSONLint」**（一個免費的語法檢查網站）。
4. 貼上你的文字並點擊驗證，它會精準告訴你「第幾行少了一個括號」。把括號補上，軟體就能順利開啟了！

---

## 🎓 給學生的防雷金句（Best Practices）

1. **圖形介面優先：** 盡量從軟體的「UI 點擊設定」來新增模型，比自己手動改設定檔安全很多。
2. **修改前先備份：** 要大改設定檔前，先把整個 `opencode` 資料夾複製一份（例如叫 `opencode_備份`）。改壞了？直接把備份蓋回去，瞬間滿血復活！
